# ビジュアルチェック — Playwright MCPによる全ページ目視QA

Playwright MCPブラウザツールを使い、全4ページをデスクトップ・モバイル両方でスクリーンショット撮影し、表示崩れやコンソールエラーを検出する。

## 引数

`$ARGUMENTS` — 省略可。ページフィルタを指定（例: "index" で index.html のみチェック）。空の場合は全4ページを対象とする。

## 対象ページ

- `index.html` — トップページ
- `jobs.html` — 求人一覧
- `interview.html` — インタビュー
- `entry.html` — 応募フォーム

`$ARGUMENTS` にページ名が指定された場合、そのページのみをチェック対象とする。

## 手順

### 1. ローカルサーバーの起動確認

- `http://localhost:8765` にアクセスして応答を確認する
- 応答がない場合、`npx http-server -p 8765` をバックグラウンドで起動する
- サーバーが起動するまで待つ

### 2. 各ページのデスクトップチェック（1440x900）

対象ページそれぞれについて:

1. ブラウザウィンドウを `1440x900` にリサイズする
2. `http://localhost:8765/{ページ名}` にナビゲートする
3. ページの読み込みを待つ
4. スクリーンショットを撮影する（ファイル名: `desktop-{ページ名}.png`）
5. コンソールメッセージを確認し、エラーがあれば記録する

### 3. 各ページのモバイルチェック（375x812）

対象ページそれぞれについて:

1. ブラウザウィンドウを `375x812` にリサイズする
2. `http://localhost:8765/{ページ名}` にナビゲートする
3. ページの読み込みを待つ
4. スクリーンショットを撮影する（ファイル名: `mobile-{ページ名}.png`）
5. コンソールメッセージを確認し、エラーがあれば記録する

### 4. モバイルメニューの確認（index.html）

index.html がチェック対象に含まれる場合:

1. モバイルサイズ（375x812）で index.html を開く
2. ハンバーガーメニューボタンをクリックする
3. メニューが開くのを待つ
4. スクリーンショットを撮影する（ファイル名: `mobile-menu-open.png`）
5. ダークネイビーのオーバーレイが正しく表示されているか確認する
6. メニュー内のリンクが正しく表示されているか確認する

### 5. 報告

以下の情報をまとめて報告する:

- **撮影スクリーンショット数**: 合計枚数と各ファイル名
- **コンソールエラー**: 検出されたエラーの一覧（ページ別）。エラーがなければ「エラーなし」と報告
- **表示の問題**: スナップショットから検出された問題（あれば）
- **モバイルメニュー**: オーバーレイの表示状態

## 注意事項

- スクリーンショットは相対パスで保存する（カレントディレクトリ基準）
- ブラウザは最後に閉じる
- コンソールエラーのうち、favicon.ico の 404 など無害なものは無視してよい

# デザイントークン — CSS変数の一覧表示と更新

`css/custom.css` の `:root` ブロックに定義されたCSS変数（デザイントークン）を一覧表示し、必要に応じて更新する。

## 引数

`$ARGUMENTS` — 省略可。

- **空の場合**: 現在のトークン一覧を表示するのみ
- **`update [変数名] [値]` の場合**: 指定した変数を更新する（例: `update --color-primary #FF0000`）

## 手順

### 1. CSS変数の読み取り

- `css/custom.css` を読み取る
- `:root { ... }` ブロック内のすべてのCSS変数（`--` で始まるプロパティ）を抽出する

### 2. カテゴリ別に整理して表示

抽出した変数を以下のカテゴリに分類して表示する:

#### カラー（Colors）
- `--color-*` で始まる変数
- `--pika*` で始まる変数（ピカチュウイエロー系）
- `--navy*` で始まる変数（ネイビー系）
- その他の色に関する変数

表示形式:
```
| 変数名 | 値 | 用途・説明 |
|--------|-----|-----------|
| --color-primary | #XXXXXX | メインカラー |
```

#### フォント（Fonts）
- `--font-*` で始まる変数

#### スペーシング・サイズ（Spacing / Sizing）
- サイズや余白に関する変数

#### トランジション（Transitions）
- `--transition*` で始まる変数

#### シャドウ（Shadows）
- `--shadow*` やボックスシャドウに関する変数

#### その他
- 上記に分類されない変数

### 3. Tailwind設定とのマッピング

- `index.html` から `<script>` タグ内の `tailwind.config` ブロックを読み取る
- Tailwind の `theme.extend` で定義されているカスタムカラーやフォントを抽出する
- CSS変数との対応関係を表示する

表示形式:
```
| Tailwindクラス | CSS変数 | 値 |
|---------------|---------|-----|
| bg-pika | var(--pika-yellow) | #XXXXXX |
```

### 4. 変数の更新（`$ARGUMENTS` に `update` が含まれる場合）

`$ARGUMENTS` が `update [変数名] [値]` の形式の場合:

1. 指定された変数名が `:root` ブロックに存在するか確認する
2. 存在しない場合はエラーを報告し、類似の変数名を提案する
3. 存在する場合:
   - 変更前の値を表示する
   - `css/custom.css` の該当行を Edit ツールで更新する
   - 変更後の値を表示する
   - 「この変数を参照している箇所」を Grep で検索し、影響範囲を報告する

### 5. 報告

#### トークン一覧表示の場合

- カテゴリ別の変数一覧（上記の表形式）
- Tailwind設定とのマッピング表
- 変数の総数

#### 更新の場合

- 変更した変数名
- 変更前 → 変更後の値
- 影響を受けるファイルと箇所の一覧
- 「変更を確認するには `/visual-check` を実行してください」と案内する

## 注意事項

- CSS変数名は `--` プレフィックスを含めて指定する（例: `--color-primary`）
- 値のバリデーションは行わない（ユーザーの意図を尊重する）
- `:root` 以外のセレクタ内で定義された変数は対象外とする
- 変更は `css/custom.css` のみに適用する（HTMLファイル内のインラインスタイルは対象外）

# ページ品質監査

指定されたHTMLページ（$ARGUMENTS または全4ページ）に対して品質監査を実施してください。

## 監査項目

### 1. HTML構造
- `<!DOCTYPE html>`, `<html lang="ja">` の存在
- セマンティックタグの使用状況（header, nav, main, section, footer, article）
- 閉じタグ漏れ・ネスト不正がないか
- 全 `<img>` に `alt` 属性があるか（装飾画像は `alt=""` + `aria-hidden="true"`）
- ID の重複がないか

### 2. CSS整合性
- `css/custom.css` の CSS変数（`:root`）と Tailwind config の色定義が一致しているか
- 使用されているCSSクラスが `custom.css` に定義されているか
- 未使用のCSSクラスがないか（主要コンポーネントのみ）
- レスポンシブ対応: モバイル/タブレット/デスクトップのメディアクエリが適切か

### 3. リンク検証
- 内部リンク（4ページ間）が正しいファイル名を参照しているか
- アンカーリンク（`#`）の参照先IDが存在するか
- 外部リンクに `target="_blank"` と `rel="noopener noreferrer"` があるか
- `img/` 内の画像パスが有効か

### 4. コンテンツ品質
- テキスト内にプレースホルダー（Lorem ipsum、「ここに〜」）が残っていないか
- 電話番号が `03-6262-9556` で統一されているか
- 店舗名・住所の表記が CLAUDE.md §7 と一致しているか

## 出力フォーマット
各項目ごとに ✅（問題なし）/ ⚠️（改善推奨）/ ❌（要修正）で評価し、修正が必要な箇所は具体的なファイル名・行番号・修正案を提示してください。

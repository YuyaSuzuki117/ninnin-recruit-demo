# ページ同期 — 共通要素の一貫性チェックと同期

index.html をマスターソースとして、全4ページの共通要素が同一であることを確認し、差異があれば修正する。

## 対象ページ

- `index.html` — マスターソース
- `jobs.html` — 同期先
- `interview.html` — 同期先
- `entry.html` — 同期先

## 同期対象の共通要素

以下の要素を index.html から抽出し、他の3ページと比較する:

1. **ヘッダー**: `<header>` ブロック全体（`.site-header` から `</header>` まで）
2. **モバイルメニュー**: `<nav id="mobile-menu">` ブロック全体
3. **フッター**: `<footer>` ブロック全体（`.site-footer` から `</footer>` まで）
4. **フローティングCTA**: `.floating-cta` ブロック全体
5. **スクロールプログレス**: `#scroll-progress` の div
6. **スキップリンク**: `.skip-link` 要素
7. **Tailwind設定**: `<script>` タグ内の `tailwind.config` ブロック
8. **Google Fonts**: `<link>` タグ（fonts.googleapis.com / fonts.gstatic.com）

## 手順

### 1. マスターソースの読み取り

- `index.html` から上記8つの共通要素をそれぞれ抽出する
- 各要素の開始行と終了行を特定する

### 2. 各ページとの比較

`jobs.html`, `interview.html`, `entry.html` それぞれについて:

1. 同じ8つの共通要素を抽出する
2. index.html の対応する要素と文字列比較する
3. 差異があれば記録する

### 3. ナビゲーションリンクの例外処理

**重要**: ナビリンクの `href` は意図的に異なる:

- index.html: `href="#section-name"` （同一ページ内アンカー）
- サブページ: `href="index.html#section-name"` （トップページへのリンク）

この差異は正常であり、同期対象から**除外**する。比較時にはナビリンクの href を正規化してから比較するか、href の差異は無視する。

### 4. 差異の修正

差異が見つかった場合:

- 差異の内容を表示する（どのページのどの要素が異なるか）
- index.html の内容をマスターとして、サブページ側を修正する
- ただし、ナビリンクの href 差異は維持する（上記の例外処理に従う）

### 5. 報告

以下の情報をまとめて報告する:

| 要素 | jobs.html | interview.html | entry.html |
|------|-----------|----------------|------------|
| ヘッダー | 同一 / 修正済み | ... | ... |
| モバイルメニュー | ... | ... | ... |
| フッター | ... | ... | ... |
| ... | ... | ... | ... |

- 修正したファイルと要素の一覧
- 既に同一だった要素の一覧
- ナビリンクの差異は「意図的差異（正常）」として報告

## 注意事項

- 修正前に必ず Read ツールでファイルを読み取る
- Edit ツールで修正する（ファイル全体の書き換えは避ける）
- 空白やインデントの差異も検出対象とする（完全一致を目指す）
- ページ固有のコンテンツ（`<main>` 内など）は同期対象外
